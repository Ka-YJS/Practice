# 객체지향 프로그래밍(OOP)

1. 객체 지향 프로그래밍(OOP : Object-Oriented Programming)이란, 말 그대로 객체를 지향하는 프로그래밍 방법을 말함
2. 객체란 우리 실생활에 존재하는 모든것으로 생각할 수 있음
3. 객체는 일반적으로 상태를 표현할 수 있고, 우리가 행동으로 실행할 수 있는 모든것들을 의미함
4. 이런 객체를 중심으로 프로그램 구조를 설계하고 프로그래밍 하는 것을 객체 지향 프로그래밍이라고함

# 클래스(Class)

## 클래스란?

1. 객체를 생성하기 위한 설명서임
2. 어떤 물건을 만들기 위한 메뉴얼이라고 생각하면 됨
3. 클래스를 기반으로 객체를 생성해야 함 -> Scanner, Random 등
4. 하나의 설명서로 여러개의 물건을 만들 수 있듯이, 자바에서는 하나의 클래스로 여러 개의 객체를 생성할 수 있음

## 클래스의 선언

1. 형식 : 접근제한자 class 클래스명{ }
2. 접근제한자 : 클래스의 접근 범위를 제한하는 키워드
    - class : class를 선언함을 뜻함
    - 클래스명 : 변수처럼 이름을 가지고, 객체를 생성할 때 사용함

## 클래스의 종류와 규칙

1. 실행용 클래스
    - 프로그램 전체에서 단 하나의 클래스로, 프로그램의 실행을 맡고 있음
    - main메서드를 갖고 있으며, 다른 클래스에서 사용하지 않음
2. 객체 생성용 클래스 : 다른 클래스에서 사용할 목적으로 선언되는 클래스임
3. 클래스 이름을 작성하는 규칙
    - 영어 대소문자를 사용할 수 있으며 보통 첫 글자는 대문자를 사용함
    - 숫자를 사용할 수 있으나 첫 글자로는 사용할 수 없음
    - 특수문자는 $,_만 가능함
    - 자바 예약어(키워드)는 사용할 수 없음
4. 주의사항
    - 프로그램을 실행했을 때, 코드를 컴파일한 결과물은 코드 파일을 각각 작성한 것과 동일하게 각 class별로 도출되어 2개가 됨
    - 파일 분리여부와 상관없이 결과물이 같기때문에 분리여부는 개발자가 원하는 대로 작성해도 무방함
    - but.추후 유지보수의 편리성과 클래스의 재사용을 고려해 하나의 파일에 한개의 클래스를 작성하는 것을 추천함

## 클래스의 구성

- 클래스를 구성하는 요소는 필드,메서드, 생성자 3가지가 있음

### 필드(Filed)
1. 객체가 가져야할 데이터의 상태를 저장하는 변수를 말함
2. 필드, 전역변수, 멤버 변수 라고 부르는데 다 같은말
3. 필드의 값을 초기화 하지 않으면 객체 생성시 자동으로 기본값으로 초기화됨
4. 필드는 클래스에 포함된 요소이자, 객체를 생성한 후 객체가 가지는 변수임
5. 따라서 객체를 생성한 후 그 객체의 필드를 사용할 수 있음

### 메서드(Method)

1. 메서드(Method)란?
    - 객체의 기능을 담당하는 중괄호({})블록을 말함
    - 특정 기능을 수행하는 코드를 따로 빼서 중괄호 안에 작성하며, 1개의 메서드는 일반적으로 1개의 기능을 수행함
    - 메서드의 선언 : 메서드는 크게 선언부(signature)와 실제 영역(body)로 구성되어 있음
    ```java
    접근제한자 반환형 메서드명(파라미터){ //머리
            작업할 내용(기능/기능을 수행하는 코드)
            return 반환값;}
    //상황에 따라서 필요하지 않다면 파라미터와  return값은 없어도 됨
    ```
    - 클래스 안에서 특정 기능을 수행하기 위해 코드들을 따로 하나의 블록으로 묶어놓은 집합을 말함
    - 필요에 따라 이 집합을 호출해 사용할 수 있음
    - 메서드명은 말그대로 메서드의 이름(첫글자는 소문자로 시작함)
    - 메서드를 호출할 때 사용
2. 메서드의 사용
    - 구현한 메서드를 사용하는 방법은 필드의 사용법과 동일함
    - 메서드를 선언한 클래스 안에서 메서드를 사용할 때는 단순히 메서드명만 호출하면 되지만, 
    다른 클래스에서 메서드를 사용하려면객체를 먼저 생성한 후 참조 변수를 이용해 그 객체의 메서드를 사용해야함
    - 개체가 존재해야 메서드도 존재하기 때문임
    - 형식 : 변수명.메서드명( ); -> 생성한 객체의 메서드 호출
3. 함수의 작동원리
    - 메서드를 호출하면 블록 안에 있는 코드들이 위에서 순차적으로 모두 실행되고 경우에 따라 실행한 결과를 호출한 곳으로 돌려줌
    - 이를 '반환한다'라고 표현하고, 반환하는 결과값을 '반환값(리턴값)'이라고 함
    - 리턴값이 있을 경우에는 리턴할 데이터의 타입이 무엇인지 메서드명( )앞에 반환 타입을 기재해줘야함
    - 반환형은 메서드가 처음부터 끝까지 수행을 마친 후에 반환해야 할 값이 있을 경우에 기입
    - int, String, boolean등 기본자료형을 포함하여 사용자가 만든 객체로도 반환이 가능함
    - 아무것도 반환하지 않을때는 void를 사용함
4. 메서드 이름 짓기
    - 메서드의 이름은 그 기능을 명확하게 설명해줄 수 있게 작성하는 것이 좋음
    - 메서드명을 작성하는 규칙 역시 변수를 작성하는 규칙과 동일함

5. 메서드 사용의 이점
    - 메서드를 구현함으로써, 같은 내용의 코드를 반복적으로 사용하는 것을 피할 수 있음
    - 반복되는 문장들을 묶어서 메서드로 작성해놓으면 필요할 때마다 재사용이 가능하기 때문임
    - 코드의 집합을 따로 분리하는것을 "모듈화"라고 함
    - 모듈화를 하면 코드를 읽을 때 가독성이 좋아지며, 프로그램을 수정할 때 더욱 빠르고 쉽게 할 수 있음
6. 메서드의 호출
    - 메서드는 다른 메서드에서 호출되어 사용됨
    - 형식 : 
    ```java
    클래스명 변수명 = new 클래스명();//-> 객체를 생성하여 변수에 담기
	변수명.메서드명();
    ```
    - 메서드는 클래스 안에서 선언되므로 메서드를 사용하기 위해서는 해당 클래스의 객체부터 생성해야 함
    - 2개 이상의 메서드 선언하기
        - 메서드는 같은 클래스에 있는 필드를 사용할 수 있음
        - 하나의 클래스에 2개 이상의 메서드를 사용하는 것 역시 가능함

### 생성자(constructor) : 메서드의 일부

1. 메서드 중 객체를 생성할 때 반드시 호출해야 하는 특수한 기능을 하는 메서드임
2. 이 메서드는 객체를 생성하면서 객체 변수를 초기화하는 역할을 하기 때문에 생성자라고 부름(객체 생성/초기화할 때는 반드시 생성자를 호출해야 함)
3. 선언(생성자의 호출)
```java
클래스명 객체명 = new 클래스명( );//이 new메서드가 생성자임
```
4. 생성자라는 메서드는 클래스명과 이름이 같아야함(생성자명은 무조건 클래스명으로 써야 함)
5. 우리는 생성자라는 메서드를 정의(선언)한적이 없음에도 호출하여 객체를 생성해왔음
 ->우리가 직접 선언하지 않아도 기본 생성자가 자동으로 생성되고 우리 눈에만 보이지 않기 때문
6. 생성자를 통해 객체 변수를 초기화 한다는 말은, 단순히 어떤 값을 초기화 한다는 뜻이 아님
7. 필드와 메서드를 호출하는 등 객체를 사용하기위해서는 객체 변수가 메모리에 올라가야 하는데, 
이렇게 메모리에 객체를 올려주는 역할을 하기 때문
8. 보충설명
    - 객체를 생성해야 하는 참조용 클래스는 모두생성자를 가지고 있음
    - 생성자는 반환값이 없지만, 반환 타입을 아예 작성하지 않음(void로도 적지 않음)
    - 생성자는 객체의 필드를 초기화하기 위한 데이터를 매개변수를 통해 전달받을 수 있음
    - 생성자의 호출 위치
        - 일반 메서드들과는 다르게, 생성자를 호출하는곳이 정해져있음
        - 생성자는 클래스를 기반으로 객체를 생성할 때, 
        객체의 초기화를 담당하는 역할을 하므로 "객체를 생성할 때만 호출할 수 있음"

### 정적 멤버와 static

1. 클래스 안에서 선언된 필드와 메서드를 클래스 멤버라고도 부름
2. 클래스에 포함된 요소라는 의미로 '멤버'라는 용어를 사용함
3. 일반적으로 각 객체가 가지게 되는 필드와 메서드를 인스턴스 멤버 라고 함
4. 모든 객체들이 공유하며 사용하는 하나의 필드와 메서드를 정적 멤버라고 부름
5. 정적멤버 : 필드와 메서드를 선언할 때 static이라는 키워드가 붙은 멤버를 말함

### static

1. 사전적으로 '고정된'이라는 뜻을 가지고 있음
2. 프로그래밍 언어에서 static은 '클래스에 고정되었다'라는 의미로 사용되고 있음
3. 쉽게 말해서 객체가 아닌 클래스에 의존적인 요소라고 생각하면 됨
4. 멤버 앞에 static 키워드를 붙히게 되면, 다른 멤버들과 달리 객체를 생성하지 않고 바로 사용할 수 있음
5. 그 이유는 객체를 생성할 때 메모리에 올라가는 것이 아니라, 
프로그램이 실행될 때 메모리에 올라가고 프로그램이 종료될 때 메모리에서 사라지기 때문임
6. 정적 멤버의 경우, 객체마다 가지는 데이터 기능이 아니기 때문에 모든 객체가 같은 값을 가져야 할 경우에 사용하는 것이 효율적임
7. 따라서 각 클래스의 멤버를 선언할 때는 충분히 고려한 후 정적 멤버로 선언할지에 대한 결정을 내리는 것이 좋음
8. 정적 멤버를 호출할 때, 객체 변수를 통해 호출할수도 있음
9. 보충설명
    - 프로그램 시작부터 종료시까지 계속 값을 유지해야하는 경우에 사용됨
    - static을 쓰면 데이터영역에 값을 생성 및 계속 유지하므로 힙영역에 따로 값을 배정
    (인스턴스변수일 경우 같은 값을 힙영역에 계속 개별적으로 복사하므로 그것을 방지하기 위함임)
    - 객체를 생성할 수 없는데, 값은 사용해야 하는 경우 사용됨 -> 객체를 생성하지않아도 호출(직접접근)이 가능함
    - but. private static으로 사용하면 getter+setter생성해서 값에 접근가능하도록 해야함

### 접근제한자

1. 접근제한자는 클래스/메서드/필드에 대한 접근을 어디범위까지 제한하느냐에 대한 지시어
2. 종류 : public, private, protected, default
3. public : 모든 접근을 허용. 같은 프로젝트 내의 모든 객체들이 사용할 수 있도록 허용
4. private : 현재 클래스 내에서만 사용이 가능함
5. protected : 상속관계의 객체들에만 사용이 가능함
6. default : 같은 패키지(폴더)내의 객체에만 사용을 허가(아무것도 쓰지 않으면 default)

### 반환형(return Type)

1. 메서드를 호출하면 메서드는 블록(중괄호) 안에 있는 코드들을 실행한 후 결과값을 반환함
2. 이때 결과값을 어떤타입으로 반환할것인지 미리 정해주는것임
3. 반환값이 없는 경우 타입으로 'void'를 쓰면 됨
4. return
    - 함수에서 모든 작업을 마치고 경우에 따라 실행한 결과를 호출한곳으로 다시 돌려주기도 하는데. 이것을 '반환한다'라고 표현함
    - 반환하는 결과값을 '반환값'이라고 부르기도 함
    - 리턴값이 있을 경우에는 리턴할 데이터의 타입이 무엇인지 반환형에 기재해줘야 함
    - 리턴값이 없는 경우 메서드를 종료하기 위해 return을 사용할 수 있음
    - 매개변수와 마찬가지로 리턴값의 자료형은 제한이 없음
    - 자바에서 사용하는 모든 자료형을 반환타입으로 사용할 수 있음
    - 형식
    ```java
    	접근제한자 반환타입 메서드명( ){
        기능을 수행할 코드들
        ...
        return 결과값;}
    ```

### 매개변수(파라미터)

1. 특정 기능을 수행하기 위한 메서드는 기능을 수행할 때 사용할 값(인수)를 전달받을 수 있음
2. 매개변수는 사용할 값을 받는 변수임
3. 형식
```java
    접근제한자 반환타입 메서드명(자료형 변수명, 자료형 변수명){
        기능을 수행할 코드}
```
4. 매개변수는 '매개변수의 자료형'과'매개 변수명'으로 선언할 수 있음
5. int number;에서 int는 전달받을 값의 자료형, number은 메서드 안에서 사용할 이름임
6. 파라미터의 개수에는 제한이 없음
7. 2개 이상의 파라미터를 정의할 때는 콤마(,)를 기준으로 변수를 여러개 만들면 됨
```java
  접근제한자 반환형 메서드명(자료형 변수명1, 자료형 변수명2, ...){  }
```
