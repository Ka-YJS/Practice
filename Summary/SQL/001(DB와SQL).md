# DB

1. 구조화된 정보, 데이터의 조직화된 모음으로 일반적으로 컴퓨터에 전자적으로 저장이 되고  데이터베이스관리시스템에 의해 제어됨
2. DBMS(DataBaseManagementSystem) : 데이터베이스와 사용자 또는 프로그램 간의 매개체 역할을 하여 사용자가 정보의 구성,검색,수정,삭제와 같은 관리를 할 수 있게 해줌

## DIKW 피라미드

1. DIKW
    - Data / Information / Knowledge / Wisdom
    - Data -> (가공) -> Information -> (활용) -> Knowledge -> (도출) -> Wisdom

## 데이터의 종류

1. 정형데이터 : 표형식으로 정리된 데이터(ex. 엑셀)
2. 비정형데이터 : 텍스트데이터, 이미지, 동영상, 음악 등(ex. 인스타게시글, 카톡, 유튜브동영상 등)
3. 반정형데이터 : 특정한 태그(부등호가 존재하는 것)(으)로 쌓여져 있는 것(ex. html, xml, css 등)

## 데이터베이스

1. 데이터들이 모여있는 데이터들의 집합
    - 공유+쳬계화+적은중복+저장 되어있어야 함
    - 컴퓨터 전원을 켜고 끌 때 저장유무에 따라 저장되면 비휘발성, 저장안되면 휘발성
2. 서로 관련이 있는 데이터들의 모임
3. '연관성'이 있어야 데이터베이스가 됨
    - 메모장에 두서없이 적은 단어들의 모임은 연관성이 없음
    - 메신저 프로그램에 등록된 이름들은 연관성이 있음 -> 정의상 데이터베이스라고 할 수 있음
4. 특징
    - 공유데이터 : 여러 사용자가 함께 접근하고 이용함
    - 통합 데이터 : 중복을 피하고, 효율적으로 이용함
    - 저장데이터 : 저장되어있고, 여러매체에서 접근할 수 있는 데이터
    - 운영데이터 : 주요기능을 수행하기위해 필요한 데이터

# DBMS

1. 데이터베이스(DB)의 정의, 갱신, 질의처리, 유지보수, 보안 등의 편리한 기능을 제공하는 시스템
2. 한마디로 DB를 관리하는 시스템이라고 할 수 있음
3. 다수의 사용자들이 데이터베이스내의 데이터에 접근할 수 있도록 해줌
4. 사용자 및 개발자 둘 다 이용함 -> 사용자는 app을 사용, 개발자는 ms/오라클 등에서 제공하는 개발용을 사용
5. 기능
    - 정의기능 : DB구조를 정의
    - 조작기능 : 데이터의 검색, 삽입, 삭제, 갱신 등의 작업을 지원
    - 제어기능 : 데이터를 정확, 안전하게 유지하는 기능
6. DBMS와 파일시스템
    - DBMS : 데이터 모델/조작, 데이터 공유, 동시성 제어, 장애복구, 보안성, 확장성
    - 파일시스템 : DBMS가 출시되기 이전에 주로 사용했던 데이터 관리 시스템
        - 파일은 디스크에 데이터/코드/텍스트 등을 저장하는 단위
        - 파일시스템은 운영체제(OS)가 기능을 담당
        - 모든 데이터베이스는 파일 형태로 디스크에 저장
        - DBMS를 사용하지않고, 응용프로그램에서 직접 데이터베이스를 관리함
7. 장점
    - 데이터 중복과 불일치 감소
    - 데이터보안
    - 질의 처리에 효율적인 저장구조
    - 백업과 복구
    - 다양한 인터페이스
    - 일관된 데이터를 유지함
8. 단점 : 자원이 많고 복잡하며, 비용(인적자원,시간)이 높음

## DBMS의 종류

1. 네트워크형, 계층형, 관계형, 객체지향형 등이 있음
2. 다양한 종류가 있으나, 관계형DBMS(RDBMS)가 가장 많이 쓰임 -> 난이도가 낮고, 편리함
3. 많이 쓰는 DBMS 솔루션 : Oracle, MS-SQL, Access, MySQL, DB2, Sybase 등

### 계층형 데이터베이스(HDBMS)

1. 폴더와 파일 등의 계층 구조로 데이터를 저장하는 방식(트리구조)
2. 데이터의 관계를 트리 구조로 정의, 부모-자식 형태를 가짐
3. 하드디스크,DVD파일 시스템 등
4. 데이터 액세스 속도가 빠름
5. 데이터의 사용량 쉽게 예측
6. 상하 종속적인 관계로 구성되어 초기 세팅 후 프로세스 수용이 어려움

### 네트워크형 데이터베이스(NDBMS)

1. 데이터 구조를 네트워크상의 노드 형태로 논리적이게 표현한 데이터 모델, 각각의 노드를 서로 대등한 관계로 구성한 시스템
2. 계층형 데이터베이스의 데이터 중복 문제 해결, 상하 종속적 관계 해결
3. 추후 변경시 복잡한 구조로 인해 변경이 어려움
4. 데이터 종속성을 해결하지 못함
5. 망(그물)의 형태를 가지며, 모든 것이 연결되어있어서 보안성이 높음

### 관계형 데이터베이스(RDBMS)

1. 행(Column)과 열(Rows)을 가지는 표 형식 데이터를 저장하는 형태의 데이터베이스
2. SQL을 이용하여 관리 및 접근
3. 다양한 용도로 사용 가능(범용성), 높은 성능
4. 데이터 무결성 보장, 높은 신뢰성
5. 데이터의 분류, 정렬, 탐색 속도가 빠름
6. 대량의 데이터 입력 처리 가능
7. 테이블형태(행+열)로 이루어져있어 다루기가 쉬움
8. 대부분의 DBMS가 RDBMS의 형태로 사용됨

### 인기있는 RDBMS 소프트웨어
1. MySQL : 오픈소스 커뮤니티에서 개발
2. MSSQL(Microsoft SQL Server) : 마이크로소프트에서 개발, 윈도우 플랫폼에서만 동작
3. PostgreSQL : 오픈소스 커뮤니티에서 개발
4. Oracle Database : 오라클에서 개발, 가장 많이 사용하는 DBMS
5. DB2 : IBM이 개발, 한동한 IBM 컴퓨터에서만 구동이 가능했다. 등등
6. MariaDB

### 데이터베이스시스템(DBS)과 사용자

1. 사용자가 DBMS를 통하여 물리적인 DB와 소통하는 형태의 시스템
2. 쉽게 정의하자면 DBS = DB + DBMS의 형태라고 볼 수 있음
3. 데이터베이스 사용자(Database User)
    - 구축된 DB를 사용하는 사람들의 총칭
    - 크게 개발자와 사용자(요청자)로 나뉘며, 사용자의 예시로는 편의점점장이나 병원직원 등이 있음
    - 세부적으로는 데이터베이스 관리자, 시스템프로그램 개발자, 업무분석가, 응용프로그램 개발자, 일반사용자 등이 있음
4. 데이터베이스 관리자(DBA)
    - 데이터베이스의 전체 환경구성과 운영에 관련된 전반적인 책임자
    - 설계에 대해 모든 책임이 있음
    - 데이터베이스 관리자는 스키마의 정의부터 접근권한까지 모든 일을 수행함
    (스키마는 Data테이블에서 속성부분이라고 할 수 있음)
5. 데이터베이스 설계자(Database Designer)
    - 요구사항을 분석해 DB의 개념적, 물리적 스키마의 설계를 책임짐(요구사항이 없으면 DB를 제작하지않음)
    - DB의 구성요소를 결정하고, 스키마를 정의함
    - 저장구조와 접근방법을 선정함
    - 효율적인DB를위해 정규화를 수행하여 사용자들의 요구를 만족시킴(데이터베이스를 만들 때 중복을 감소시킴)
6. 최종 사용자(End User)
    - 일반사용자, 초보사용자, 전문사용자 등으로 나뉨
    - 의뢰자라고도하며, 프로그램을 사용만하기때문에 프로그램이나 코드를 작성하지않음
    - 보통 DB에대한 질의 및 갱신 연산을 주로하는 단순 사용자
    - DBMS를 이용함
7. 시스템 분석가/응용프로그램개발자(System Analyst/Application Programmer)
    - 초보사용자를 위하여 잘 정의된 기능의 응용프로그램을 설계 및 구현하는 사람
    - 시스템 분석가 : 사용자의 요구사항을 분석하고 사용자를 만족시킬 수 있는 미리 작성된 트랜잭션들의 명세를 설계함
    - 응용프로그램 개발자
        - 사용자를위한 프로그램 설계 및 구현을 담당함
        - 트랜잭션들의 명세를 이용하여 프로그램 구현, 테스트, 오류수정, 문서화 등의 작업을 함
        - 미리 작성된 트랜잭션들의 유지 및 관리를 함
        - 일반적인 개발자이며, 사용자를 위한 App을 개발함
        - 트랜잭션 : DB에서 사용되는 최소업무처리의 단위(연산단위)

# SQL

### SQL문

1. 구조화된 질의언어(Structured Query Language)의 약자로 DBMS 상에서 데이터를 읽고 쓰고 삭제하는 등 데이터를 관리하기 위한 일종의 프로그램 언어임
2. C나 Java같은 일반 프로그래밍 언어는 절차적 언어라고 할 수 있는데 반해 SQL은 집합적 언어임
3. 집합적 언어는 데이터를 특정 집합 단위로 분류해 이 단위별로 한 번에 처리하는 언어임(일반적인 프로그래밍 언어는 변수를 다루며, 개발+알고리즘을 사용함)
4. RDBMS의 표준 언어로 SQL을 채택했고, 표준 SQL문만 학습하면 오라클은 물론 MSSQL, DB2등 여러 DBMS에서  동일하게 사용할 수 있다는 뜻임. 물론 DBMS별로 독특한 기능이 있지만, 다른 프로그래밍 언어에 비해 그 차이는 미미함
5. DB에서 데이터를 저장, 업데이트, 검색, 제거 등을 할 수 있음
6. DB성능을 유지관리하고, 비휘발성 데이터를 다루며, 최적화하는데 SQL이 사용됨
7. SQL문장은 대소문자를 구별하지 않음
8. 한줄 또는 여러줄에 걸쳐 입력하는 것이 가능
9. SQL문장의 끝은 세미콜론(;)으로 맺어야 함

## 데이터베이스 언어

1. 데이터 정의어(DDL, Data Definition Language)
    - 데이터간의 관계를 정의하거나 이미 정의된 데이터베이스의 구조를 변경하거나 수정하는데 사용되는 언어로, DB와 Table을 다룸
    - DDL : C_CREATE, D_DROP, A_ALTER, T_TRUNCATE, R_RENAME
    - DDL + 추가
        - SI(시스템 통합) : 고객으로 부터 수주받아서 만드는 것 -> 0에서부터 쌓아올림(고객사쪽으로 파견)
        - SM(시스템 관리) : 이미 만들어진 프로젝트를 유지보수
2. 데이터 조작어(DML, Data Manipulation Language)
    - 저장된 실제 데이터를 검색, 수정, 삭제, 삽입 등을 수행하는데 사용하는 언어 -> 실제로 데이터를 조작함
    - 개발자 입장에서 가장 많이 사용하는 SQL
    - DML : S_SELECT, I_INSERT, U_UPDATE, D_DELETE

3. 데이터 제어어(DCL, Data Controll Language)
    - 관리자가 데이터를 관리하려고 데이터의 보안, 무결성, 데이터의 복구, 병행수행 제어 등을 정의할 때나, 사용자의 권한(트랜잭션도 함)을 설정할 때 사용하는 언어
    - DML : G_GRANT, R_REVOKE
4. 트랜잭션 제어어(TCL, Transaction Controll Language) 
    - TCL : C_COMMIT, R_ROLLBACK, S_SAVEPOINT

### DBMS에서 SQL실행방법

1. 구문분석(일종의 번역)을 통해서 SQL을 해석함
2. 대소문자 및 띄어쓰기를 번역 -> 문법에 오류가 없는지 확인 -> SQL문을 수행하기위한 요소들이 존재하는지 확인 등을 진행
3. 최종적으로 구문 분석이 완료되면 명령을 실행함
4. 한마디로 번역을 통해서 DBMS가 인식(전달)을 하는 것이라고 할 수 있음
